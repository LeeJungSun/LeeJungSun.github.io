Live Region

페이지가 로드 되지 않고 사용자 포커스가 다른 곳에 있을 때 컨텐츠 내용이 동적으로 업데이트되는 영역으로 Live Region의 사용 예로는 주식 시세, 채팅, 에러, 타이머, 검색창 텍스트 자동 완성 등이 있습니다.

이러한 영역은 사용자 포커스가 다른 곳에 존재할 때 컨텐츠가 업데이트 되기 때문에 보조기술 사용자는 동적으로 업데이트되는 내용을 인식하기 어렵습니다.

Live Region은 보조기술 사용자에게 현재 포커스를 잃지 않고 동적으로 업데이트되는 내용을 제공하는 역할을 합니다.

WAI-ARIA 역할(Roles), 속성(Properties) 및 상태(States)

  • role="alert"
    사용자의 작업을 방해하지 않고 간략하고 중요한 메시지를 표시하는 요소입니다.
    동적으로 렌더링 된 Alert은 대부분의 보조 기술에서 자동으로 사용자에게 메시지를 알리고 일부 운영체제에서는 경고음을 울릴 수 있습니다.
    NOTICE :
    • 자동으로 사라지는 Alert을 디자인하지 않는 것이 중요합니다. 너무 빨리 사라지는 경고는 WCAG 2.0 / 2.2.3 항목을 충족시키지 못할 수 있습니다.
    • Alert으로 사용자에게 빈번한 중단을 주지 않아야 합니다. Alert으로 인한 빈번한 중단은 시각 및 인지 장애를 가진 사람들의 유용성을 저해하여 WCAG 2.0 / 2.2.4의 요구 사항을 충족시키지 못하게 합니다.
    • 사용자 인터렉션 요소를 포함해서는 안됩니다. role="alert"은 포커스가 이동할 필요가 없기 때문에 콘텐츠 작업자는 인터렉션 요소를 제공해야 할 경우 role="alertdialog"를 사용해야 합니다.
  • aria-live
    사용자가 탐색하고 있는 영역와 상관없이 새롭게 업데이트된 정보를 사용자에게 즉시 알리는 영역을 의미합니다.
    [Values]
    • off (default)

      해당 영역을 사용자가 탐색하는 중이 아닐 경우 변경사항을 사용자에게 제공하지 않습니다.

    • assertive

      사용자가 진행하던 작업을 중단하고 변경사항을 사용자에게 즉시 알리도록 하는 역할을 합니다.
      페이지에 오류가 발생했을 경우나 사용자가 작업한 입력란이 업데이트 되는 경우와 같이 중요하거나 긴급하게 알려야 할 경우 사용하는 것을 권고합니다.

    • polite

      사용자가 진행하던 작업을 마치면 변경사항을 알리도록 하는 역할을 합니다.
      중요하지만 긴급하지 않은 변경 사항일 경우에 적합합니다.

  • aria-atomic
    변경사항을 사용자에게 전달할 때 변경된 영역만 제공할지 변경된 영역 포함하여 전체 영역을 제공할지 여부를 나타냅니다.
    [Values]
    • false (default)

      변경된 사항만 표시합니다.

      '보통' 텍스트가 변경될때 변경된 텍스트만 불러줍니다.

      <p role="alert" aria-live="polite" aria-atomic="false">비밀번호 안전성 : <span>보통</span></p>

      NVDA 음성출력 뷰어 예시)
      경고 비밀번호 안전성 : 위험
      보통

    • true

      변경된 사항을 포함한 전체 영역을 표시합니다.

      '보통' 텍스트가 변경될 경우 role="alert"이 선언된 태그 내 텍스트를 모두 읽어줍니다.

      <p role="alert" aria-live="polite" aria-atomic="true">비밀번호 안전성 : <span>보통</span></p>

      NVDA 음성출력 뷰어 예시)
      경고 비밀번호 안전성 : 위험
      경고 비밀번호 안전성 : 보통

  • aria-relevant
    라이브 영역에서 콘텐츠의 업데이트 유형(추가, 제거 등)의 정보를 알릴지 여부를 설정하는데 사용되는 선택적 속성입니다.
    NOTICE :
    • 값은 한가지 이상 설정할 수 있습니다. (기본값 : aria-relevant="additions text")
    • 설정된 값과 관련이 없는 변경은 aria-live="off"값으로 설정된 경우와 동일한 방식으로 작동합니다.
    [Values]
    • additions

      요소 노드가 라이브 영역 내 DOM에 추가되었을때 알립니다.

    • removals

      요소나 텍스트가 라이브 영역 내 DOM에 제거되었을때 알립니다.

    • text

      텍스트가 라이브 영역 내 DOM에 추가되거나 제거되었을때 알립니다.

    • all

      "additions removals text"을 모두 사용한 것과 같습니다.

  • aria-busy
    [Values]
    • false (default)

      요소에 예상되는 컨텐츠 업데이트가 없습니다.

    • true

      요소가 업데이트 중입니다.

Live Region 사용 예시 컨텐츠

Live Region을 사용할 수 있는 컨텐츠들 중에 많이 사용되거나, 이슈사항이 있었거나, 새롭게 알게된 예시를 기준으로 선정하여 테스트해보았습니다.

1. form alert

form 입력 후 유효성 검증 단계에서 오류가 있을 경우 사용자에게 오류 내용을 알려주는 역할을 합니다.

유효성 검증 단계에서 사용자에게 알려주는 방식은 크게 두 가지 방식이 있습니다.

  • form 제출 완료 후 검증 방식

    사용자가 전체 form 요소를 입력한 후 제출하였을 때 한 번에 검증하여 오류 내용을 사용자에게 제공해줍니다.
    이때, 오류 항목이 있는 경우 해당 항목의 인풋으로 포커스를 이동해주어야 사용자에게 더 나은 환경을 제공해줄 수 있습니다.

    테스트 코드 보기
  • form 요소 타이핑 중 검증 방식

    사용자가 form 요소에 내용을 입력하는 동안 즉시 검증하여 양식을 제출할 필요 없이 오류 내용을 알려줍니다.

    테스트 코드 보기

2. 검색어 자동완성

aria-live="polite"를 사용하여 사용자가 검색어 입력을 마치면 총 몇 개의 자동완성 결과가 있는지 제공해주는 역할을 합니다.

참고 URL : https://www.daum.net

<span aria-live="polite">4개의 자동완성 결과가 있습니다. 위/아래 방향키를 눌러서 이용하세요</span>

3. 타이머

초 단위로 카운트되는 타이머의 경우 해당 영역에 접근하지 않은 상태인데도 사용자에게 지속적으로 제공될 경우 사용성을 저해할 수 있으므로 aria-live 값을 제공하지 않는 것이 좋습니다.

  • aria-live 선언하지 않은 경우

    해당 타이머 영역에 접근할 때 한 번만 값을 제공해줍니다.

    테스트 코드 보기
  • aria-live 선언 한 경우 (default 값 제외)

    해당 영역에 있지 않아도 타이머가 변경될 때마다 변경된 값을 알려줍니다.

마치며

제가 이번 블로깅을 통해서 느낀 점이 총 두가지가 있는데요.

첫번째로는 내가 직접 경험하는 것 입니다. 그냥 단순히 속성들을 알아보고 다양한 예제들을 보기만 하는 것 보다, 내가 직접 해당 속성을 사용해보고 테스트 코드로 만들어보고, 스크린리더로 들어보는 것과 같이 직접 경험해야 습득도 빠르고 기억에 더 남았어요 특히 타이머는 업무 중에 이슈가 있었던터라 더욱 기억에 남았습니다.

두번째로는 스크립트 스킬(!) 입니다. 상황, 속성에 따라 속성값이 동적으로 변경되는 경우도 있고 또 내가 원하는대로 적용해보려면 스크립트를 어느정도 할 줄 알아겠구나.. 라는 생각이 들었습니다ㅠㅠ…

자료를 탐색하고 정리하는 부분도 어려웠지만 직접 테스트코드를 만들어보고 테스트하고 습득하고 이해하는 과정이 제일 어려웠던 것 같아요.

많이 부족한 글 봐주셔서 감사합니다 :)
